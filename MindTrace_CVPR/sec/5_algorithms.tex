\section{Classification Algorithms}
\label{sec:algorithms}

This study implemented and compared four classification models selected to represent both traditional machine learning and deep learning approaches under identical experimental conditions.

\subsection{Support Vector Machine (SVM)}

SVM served as a strong baseline classifier by maximising the margin between classes~\citep{dzikovska2012using}. The optimisation objective is defined as:
\begin{equation}
\min_{w, b} \;\frac{1}{2}\|w\|^2 + C \sum_{i} \xi_i
\label{eq:svm}
\end{equation}
subject to $y_i(w \cdot x_i + b) \geq 1 - \xi_i$ and $\xi_i \geq 0$, where $w$ is the weight vector, $b$ is the bias term, $C$ is the regularisation parameter, and $\xi_i$ are slack variables permitting soft-margin classification.

\subsection{Extreme Gradient Boosting (XGBoost)}

XGBoost was applied as an ensemble learning method that minimises a regularised objective function~\citep{chen2016xgboost}:
\begin{equation}
\mathcal{L} = \sum_{i} l(y_i, \hat{y}_i) + \sum_{k} \Omega(f_k)
\label{eq:xgboost}
\end{equation}
where $l$ is the loss function measuring prediction error, $\hat{y}_i$ is the predicted value, and $\Omega(f_k)$ is the regularisation term penalising model complexity.

\subsection{Convolutional Neural Network (CNN)}

CNN models extracted local n-gram features through convolution and pooling layers~\citep{hasan2019emotion}. For an input sequence $x_{1:n}$, a filter $w \in \mathbb{R}^{k \times d}$ of width $k$ produces a feature map, and max-over-time pooling extracts the most salient feature:
\begin{align}
c_i &= \text{ReLU}(w \cdot x_{i:i+k-1} + b) \label{eq:conv}\\
\hat{c} &= \max_{i}\, c_i \label{eq:maxpool}
\end{align}
where $b$ is a bias term and $x_{i:i+k-1}$ denotes concatenated embeddings over a $k$-word window.

\subsection{Bidirectional LSTM (BiLSTM)}

BiLSTM captured bidirectional contextual dependencies in sequential text by processing token sequences in both forward and backward directions~\citep{sun2023bidirectional}. Each LSTM unit computes:
\begin{align}
f_t &= \sigma(W_f [h_{t-1}, x_t] + b_f) \label{eq:forget}\\
i_t &= \sigma(W_i [h_{t-1}, x_t] + b_i) \label{eq:input}\\
\tilde{c}_t &= \tanh(W_c [h_{t-1}, x_t] + b_c) \label{eq:candidate}\\
c_t &= f_t \odot c_{t-1} + i_t \odot \tilde{c}_t \label{eq:cell}\\
o_t &= \sigma(W_o [h_{t-1}, x_t] + b_o) \label{eq:output_gate}\\
h_t &= o_t \odot \tanh(c_t) \label{eq:hidden}
\end{align}
where $f_t$, $i_t$, $o_t$ are the forget, input, and output gates respectively, $c_t$ is the cell state, $\sigma$ is the sigmoid function, and $\odot$ denotes element-wise multiplication. The bidirectional variant concatenated forward and backward hidden states $h_t = [\overrightarrow{h_t}; \overleftarrow{h_t}]$ to capture both past and future context at each time step.

\subsection{Evaluation Framework}

All models were evaluated under identical experimental conditions. Macro-averaged F1 (equal class weighting, sensitive to minority class performance) was the primary evaluation metric, ensuring that all six emotion classes contribute equally regardless of prevalence. Confusion matrices provided class-wise predictive analysis, with particular focus on semantically similar emotion pairs (fear/surprise; sadness/love). The 80/20 stratified train-test split preserved class proportions across all subsets, ensuring consistent class representation throughout training and evaluation.

\subsection{Deployment Architecture}

Following evaluation, the best-performing classical model was serialised and integrated into a lightweight REST API deployment pipeline. The SVM model (along with its TF-IDF vectoriser) was packaged using Flask and containerised with Docker to ensure reproducibility. The inference endpoint accepts raw text input, applies the same preprocessing pipeline used during training, and returns a predicted emotion label with class probability scores. Figure~\ref{fig:deployed_app} shows the deployed web interface.

\textbf{CI/CD Pipeline.} A GitHub Actions workflow automates the full build-and-deploy cycle. On every push to the \texttt{main} branch, the pipeline executes two sequential jobs: (1)~\emph{Lint \& Smoke Test} --- installs dependencies, runs Flake8 on critical error codes, and verifies that the serialised model loads correctly; (2)~\emph{Build \& Push} --- builds the Docker image via Docker Buildx and pushes it to Docker Hub (\texttt{yolandalim/125970-mindtrace}) tagged with both \texttt{latest} and the commit SHA. Pull requests trigger only the lint-and-test stage, preventing untested code from being deployed. This automated pipeline ensures that every production deployment is reproducible and regression-tested.
